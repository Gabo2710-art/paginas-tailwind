Tipos de Atributos



Los atributos son elementos importantes que permiten almacenar los diferentes datos de las entidades. Además éstos pueden ser de diferentes tipos dependiendo de sus características, lo que conlleva a un mayor abanico de posibilidades a la hora de definirlos.

Los tipos más comunes de atributos son los que se mencionan a continuación:


  • Atributos Simples: Son los mas comunes y se usan para definir las características de las entidades.

    Simbología:



  • Atributos Derivados: Su valor se puede calcular a partir de otros atributos. Ejemplo: la edad de una persona calculada a partir de su fecha de nacimiento.

    Simbología:



  • Atributos Clave: Identifican de manera única a una entidad en una base de datos. Son muy importante porque nos permite reconocer a una tupla de otra. Ejemplo: el número de identificación de un empleado.

    Simbología:




  • Atributos Multivaluados: Podemos insertar un atributo multivaluado que se representa con doble circulo y lo que nos esta diciendo el diseñador de la base de datos, es que este atributo puede replicarse.

    Simbología:




  • Dominio: Un dominio describe un conjunto de posibles valores para cierto atributo. Como un dominio restringe los valores del atributo, puede ser considerado como una restricción. Matemáticamente, atribuir un dominio a un atributo significa "todos los valores de este atributo deben de ser elementos del conjunto especificado".



    Existen varios tipos de dominios en una base de datos. Algunos de los más comunes son:


    • Dominio numérico: Este tipo de dominio incluye valores numéricos, como enteros, decimales o números reales. Por ejemplo, el dominio de un campo que registra la altura de una persona podría ser un número decimal.

    • Dominio de texto: Aquí, los valores son cadenas de texto, como nombres, direcciones o descripciones. Un dominio de texto podría incluir palabras, frases o incluso párrafos.

    • Dominio de fecha y hora: Este tipo de dominio se utiliza para representar fechas y horas. Puedes encontrarlo en campos que almacenan información como la fecha de nacimiento de una persona o la hora de una transacción.

    • Atributos Booleanos: En este caso, los valores son verdaderos o falsos. Es útil para campos que requieren respuestas binarias, como «sí/no» o «verdadero/falso».
